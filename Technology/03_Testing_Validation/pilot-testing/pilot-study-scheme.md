# Pilot Study Scheme - Online Testing

**Date:** [To be scheduled]  
**Duration:** Approximately 60 minutes  
**Format:** Online via Zoom  
**Platform:** PoetryAI Website - https://poetry.aitutor.ink/

---

## Overview

This pilot study will test the PoetryAI chatbot system with 4 individual participants (T1, T2, T3, T4) in separate testing rooms. Each participant will evaluate different aspects of the system including login methods, chatbot functionality, creative guidance, and chat history features.

## Testing Protocol

### Phase 1: Introduction & Setup (10 minutes)

**Program Director Actions:**
- Welcome all participants in Zoom session
- Provide individual instructions to each participant (T1, T2, T3, T4)
- Explain testing objectives and expectations
- Share website link: https://poetry.aitutor.ink/
- Answer any initial questions

**Participant Actions:**
- Access the PoetryAI website
- Review the tutorial section
- Familiarize themselves with the interface

---

### Phase 2: Login Testing (5 minutes)

**All Participants (Email/Password Login):**
1. Navigate to login page
2. Enter provided email address
3. Enter provided password
4. Test login functionality
5. Note any issues or observations

**Room Assignment:**
- T1 enters Room A
- T2 enters Room B
- T3 enters Room C
- T4 enters Room D

---

### Phase 3: Chatbot Functionality Testing (20 minutes)

**Testing Focus Areas:**

#### 3.1 Conversation Flow
Participants should evaluate:
- How natural the conversation feels
- Whether the chatbot responds appropriately to inputs
- If the conversation progresses logically
- Response time and system performance
- Any interruptions or errors in the flow

#### 3.2 Chatbot Functions
Participants should test:
- Basic text input and output
- Command recognition
- System prompts and guidance
- Error handling
- Special features or tools provided

#### 3.3 Creative Guidance
Participants should assess:
- Does the chatbot encourage creative thinking?
- Does it provide helpful suggestions for poetry writing?
- Does it guide users through the creative process?
- Are the prompts stimulating and thought-provoking?
- Does it help users explore different poetic approaches?

#### 3.4 Inspiration & Motivation
Participants should evaluate:
- Does the chatbot inspire new ideas?
- Is the interaction motivating?
- Does it build confidence in poetry writing?
- Are the responses encouraging?
- Does it spark imagination and creativity?

**Testing Instructions:**
- Interact with the chatbot naturally as you would when writing poetry
- Try different types of requests and inputs
- Test both simple and complex interactions
- Note what works well and what doesn't
- Pay attention to your emotional response to the interaction

---

### Phase 4: Chat History Function Testing (15 minutes)

#### 4.1 Viewing Chat History
**Test Objective:** Verify that all conversation history is accessible

**Testing Steps:**
1. Navigate to the 'History' section
2. Check if all previous conversations are displayed
3. Verify conversations are organized clearly
4. Check if timestamps are accurate
5. Test scrolling through multiple chat entries (if applicable)

**Evaluation Criteria:**
- ✓ Can see all chat history
- ✓ History is organized and easy to navigate
- ✓ Timestamps are visible and correct
- ✓ No missing conversations
- Note any display issues or missing features

---

#### 4.2 Renaming Chat History
**Test Objective:** Verify ability to rename individual chat sessions

**Testing Steps:**
1. Select an individual chat history entry
2. Locate the rename function (button/icon/menu)
3. Click on rename option
4. Enter a new name for the chat session
5. Save the new name
6. Verify the name change is displayed correctly
7. Refresh the page and check if the name persists

**Evaluation Criteria:**
- ✓ Rename function is easy to find
- ✓ Rename process is intuitive
- ✓ New name is saved successfully
- ✓ Name persists after page refresh
- Note any difficulties or bugs encountered

---

#### 4.3 Downloading Chat History as PDF
**Test Objective:** Verify ability to download chat history in PDF format

**Testing Steps:**
1. Select a chat history entry to download
2. Locate the download/export function
3. Click on download option
4. Select PDF format (if multiple formats available)
5. Initiate the download
6. Open the downloaded PDF file
7. Verify content is complete and properly formatted

**Evaluation Criteria:**
- ✓ Download function is accessible
- ✓ PDF is generated successfully
- ✓ Content is complete and readable
- ✓ Formatting is appropriate
- ✓ File opens without errors
- Note any issues with content, formatting, or download process

---

Participants will complete their assigned reflection template (see separate documents below).

## Technical Support Protocol

**If participants encounter issues:**
1. Use Zoom chat to report problems immediately
2. Program director provides real-time support
3. Document all technical issues in detail
4. Continue testing other functions if one feature fails
5. Note workarounds or solutions attempted

## Data Collection Points

- [ ] Login success rate (email/password vs OTP)
- [ ] Time taken for each testing phase
- [ ] Number and type of errors encountered
- [ ] Participant feedback on conversation flow
- [ ] Ratings on creative guidance and inspiration
- [ ] Chat history function success rate
- [ ] Reflection form responses
- [ ] Screen recordings (if consent given)
- [ ] Zoom chat logs of issues reported

---

## Post-Testing Actions

1. **Immediate:**
   - Collect all reflection forms
   - Thank participants for their time
   - Distribute participation rewards

2. **Within 24 hours:**
   - Compile technical issues log
   - Review all reflection responses
   - Identify critical bugs or problems
   - Create prioritized fix list

3. **Within 1 week:**
   - Analyze testing data
   - Prepare pilot testing report
   - Plan system improvements
   - Schedule fixes with development team

---

## Success Criteria

The pilot test will be considered successful if:
- ✓ All 4 participants can successfully login
- ✓ All can access their assigned rooms
- ✓ Chatbot responds appropriately to basic interactions
- ✓ Chat history is visible and accessible
- ✓ At least 3/4 participants can rename chat history
- ✓ At least 3/4 participants can download PDF successfully
- ✓ Major bugs are identified and documented
- ✓ Participants provide useful feedback for improvements
